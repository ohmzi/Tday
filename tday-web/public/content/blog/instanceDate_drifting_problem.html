instanceDate is the occurrence anchor that ties a generated recurring task to its override row. The drifting bug appears when the override still points at an old occurrence after the parent series has been shifted to a new sequence.

The result is familiar: the moved override stays behind on its old anchor, a new generated occurrence appears in the shifted series, and the user sees what looks like a duplicate task.

1. What goes wrong

  1. A recurring todo generates an occurrence on a specific day.
  2. The user edits that one occurrence, so an override row is stored.
  3. The parent todo later changes its dtstart or RRULE.
  4. The stored override still references the old anchored occurrence, even though the series now generates a different sequence. 
before shift: Tue Jan 20 is the anchored occurrence
override: recurId = 2026-01-20T00:00:00Z
after shift: the series now produces Wed Jan 21 instead
result: Jan 19 override remains, Jan 21 generates again

2. Root cause

instanceDate is static, but the recurrence sequence is not. If the series is re-anchored, every stored override still points at the old timeline unless something explicitly remaps it.

  • The override row is still valid as data.
  • The recurrence sequence is also valid after the series edit.
  • The bug lives in the mismatch between the old occurrence identity and the newly generated occurrence order.

3. Safer backend direction

The Ktor backend should own recurrence expansion and any sequence reconciliation logic. When a parent recurrence changes materially, it needs one of two explicit strategies:

  • Re-map existing overrides onto the new sequence using deterministic occurrence indexing or another canonical strategy.
  • Mark the old overrides as incompatible and require the client to recreate them against the new sequence.

4. Takeaway

Drifting is a data-identity problem, not just a UI bug. The clean long-term fix is to keep all recurrence interpretation in Kotlin, define one canonical occurrence identity model, and make every client consume the same reconciled result instead of each surface trying to infer it independently.
